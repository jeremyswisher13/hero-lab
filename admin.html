Analytics provider: GoatCounter

This site is wired for GoatCounter — a free, open-source, privacy-friendly analytics service that doesn’t require a cookie banner (no personal data collected, no fingerprinting). To activate, you need to claim a free GoatCounter account and update one placeholder.

Step 1. Go to goatcounter.com/signup and create a free account. Choose a code like herolab. Your dashboard URL will be https://herolab.goatcounter.com.

Step 2. In index.html and every other production HTML page, find the comment block <!-- Analytics: GoatCounter --> at the bottom of <body>. Uncomment the <script> tag and replace YOUR-CODE with the code you chose (e.g., herolab).

Step 3. In your GoatCounter settings, under “Site settings” → “Site access”, either: (a) make the dashboard public so it can be embedded below, or (b) keep it private and use the “Quick access” links below to view it in a new tab.

Step 4. In this file (admin.html), find the comment <!-- Embed: replace src --> and update the iframe’s src attribute to https://YOUR-CODE.goatcounter.com (use the same code from Step 1).

Why GoatCounter (vs. Google Analytics)

For a small research lab site, GoatCounter is a better fit than GA4 for three reasons:

  1. No cookie banner required. GoatCounter doesn’t set cookies and doesn’t collect IP addresses, so GDPR/CCPA consent isn’t needed. Academic visitors from the EU won’t hit a pop-up.
  2. Free for non-commercial use. Personal/nonprofit sites are free indefinitely. No 14-month data retention limit like GA4.
  3. Simple dashboard. One page that shows visitors, top pages, referrers, locations. No 50-menu GA4 maze.

If you ever want richer analytics (event tracking, conversion funnels, custom reports), I can swap to Plausible Analytics ($9/mo, also privacy-friendly) or wire up GA4 with a Cookiebot banner. Just say the word.
